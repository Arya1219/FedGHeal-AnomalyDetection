# Dataset

This work uses the **Server Machine Dataset (SMD)**.

> Su, Y., Zhao, Y., Niu, C., Liu, R., Sun, W., & Pei, D. (2019).  
> *Robust Anomaly Detection for Multivariate Time Series through Stochastic Recurrent Neural Networks.*  
> KDD 2019, pp. 2828–2837.

---

## Download

The dataset is not included in this repository due to size constraints.

**Source:** https://github.com/NetManAIOps/OmniAnomaly

```bash
git clone https://github.com/NetManAIOps/OmniAnomaly.git
cp -r OmniAnomaly/ServerMachineDataset/ data/raw/
```

---

## Structure

After downloading, your `data/` directory should look like this:

```
data/
├── raw/
│   └── ServerMachineDataset/
│       ├── train/                  # 28 .txt files, one per machine
│       │   ├── machine-1-1.txt
│       │   ├── machine-1-2.txt
│       │   └── ...
│       ├── test/                   # 28 .txt files
│       │   ├── machine-1-1.txt
│       │   └── ...
│       └── test_label/             # 28 binary label files
│           ├── machine-1-1.txt
│           └── ...
└── processed/                      # Generated by preprocessing scripts
    ├── graphs/                     # Serialized PyG Data objects per machine
    └── splits/                     # Train/test node index splits
```

## Dataset Statistics

| Property | Value |
|---|---|
| Machines (FL clients) | 28 |
| Sensor channels (features) | 38 |
| Total timesteps | 708,420 |
| Min anomaly rate | 0.4% |
| Max anomaly rate | 15.7% |
| Mean anomaly rate | 4.2% |
| Train / Test split | 60% / 40% |

Each machine belongs to one of three groups: `machine-1-x`, `machine-2-x`, `machine-3-x`.  
Each row in a `.txt` file is one timestep with 38 comma-separated sensor readings.  
Labels are binary: `0` = normal, `1` = anomalous.

## Preprocessing

Once raw files are in place, run:

```bash
python src/data/preprocess.py --data-dir data/raw/ServerMachineDataset --out-dir data/processed
```

This will:
1. Load each machine's train/test split
2. Construct the adaptive mutual KNN graph (k=3, RBF-weighted edges)
3. Serialize PyTorch Geometric `Data` objects to `data/processed/graphs/`
4. Save train/test node index splits to `data/processed/splits/`
